## 1. SQL Injection이란 무엇인가?

SQL Injection은 **사용자의 입력값을 통해 악의적인 SQL을
주입(Injection)** 하여\
데이터베이스를 비정상적으로 조작하는 공격 기법이다.

> "사용자가 입력한 값이 그대로 SQL 문에 포함될 때 발생하는 보안 취약점"

공격자는 이를 이용해 다음과 같은 행동을 할 수 있다:

-   인증 우회 (로그인 우회)
-   전체 데이터 조회
-   데이터 삭제
-   관리자 권한 탈취
-   테이블 구조 파괴

------------------------------------------------------------------------

## 2. 왜 발생하는가?

핵심 원인:

> SQL 문을 문자열로 직접 조합하기 때문

### 취약한 코드 예시

``` java
String sql = "SELECT * FROM users WHERE id = '" + userInput + "'";
```

사용자가 다음과 같이 입력하면:

    ' OR '1'='1

실행되는 SQL:

``` sql
SELECT * FROM users WHERE id = '' OR '1'='1'
```

→ 항상 참(TRUE) → 로그인 우회 가능

------------------------------------------------------------------------

## 3. 공격 유형

### 3-1. 인증 우회

입력값:

    admin' --

→ 비밀번호 조건이 주석 처리되어 로그인 성공

------------------------------------------------------------------------

### 3-2. UNION Injection

    ' UNION SELECT id, password FROM users --

→ 다른 테이블 데이터까지 조회 가능

------------------------------------------------------------------------

### 3-3. 데이터 삭제

    '; DROP TABLE users; --

→ 테이블 삭제 가능

------------------------------------------------------------------------

## 4. 방어 방법

### 4-1. PreparedStatement 사용 (가장 중요)

``` java
PreparedStatement pstmt = conn.prepareStatement(
    "SELECT * FROM users WHERE id = ?"
);
pstmt.setString(1, id);
```

✔ SQL 구조와 데이터 분리\
✔ 입력값이 SQL로 해석되지 않음

------------------------------------------------------------------------

### 4-2. ORM 사용 시 바인딩

MyBatis:

``` xml
SELECT * FROM users WHERE id = #{id}
```

 `${id}` 사용 금지 (문자열 치환 → 위험)

------------------------------------------------------------------------

### 4-3. 입력값 검증

``` java
if (!id.matches("^[a-zA-Z0-9]{4,12}$")) {
    throw new IllegalArgumentException();
}
```

------------------------------------------------------------------------

### 4-4. 최소 권한 원칙

-   DROP 권한 제한
-   운영/개발 계정 분리

------------------------------------------------------------------------

### 5. 에러 메시지 노출 금지

사용자에게 SQL 에러를 직접 노출하지 않는다.

------------------------------------------------------------------------

## 최종 정리

> SQL Injection은 사용자의 입력값이 SQL에 그대로 포함되어 발생하는 보안
> 취약점이며, PreparedStatement를 통해 SQL 구조와 데이터를 분리하여
> 방어할 수 있습니다.

------------------------------------------------------------------------

## 한 줄 요약

> 문자열 SQL 조합이 원인이고, PreparedStatement가 핵심 방어책이다.
