Spring 기반 애플리케이션에서 데이터베이스를 다루다 보면,  
CRUD 코드와 Repository 구현이 반복적으로 작성되는 문제를 자주 경험하게
된다.

Spring Data는 이러한 반복적인 데이터 접근 코드를 줄이고,\
Repository 계층을 표준화하여 개발 생산성을 높이기 위해 만들어진
프로젝트이다.

개발자는 비즈니스 로직에 집중할 수 있고,\
데이터 접근 로직은 Spring Data가 대신 처리해주는 구조를 제공한다.

------------------------------------------------------------------------

## 1. Spring Data가 등장한 배경

전통적인 JDBC 또는 MyBatis 기반 개발 방식에서는 다음과 같은 문제가
존재했다.

-   CRUD SQL 반복 작성
-   DAO / Mapper 클래스 반복
-   공통 코드 증가
-   유지보수 비용 증가

Spring Data는 이러한 문제를 해결하기 위해 등장하였다.

------------------------------------------------------------------------

## 2. Spring Data의 핵심 개념

### Repository 추상화

인터페이스만 정의하면 자동 구현된다.

### 공통 인터페이스 제공

CrudRepository, JpaRepository 등 제공

### 저장소 기술 독립성

JPA, MongoDB, Redis 등 지원

------------------------------------------------------------------------

## 3. 핵심 제공 기능

### CRUD

save, findById, findAll, delete 등

### Query Method

메서드 이름 기반 쿼리 생성

### Paging / Sort

Pageable, Sort 지원

### @Query

커스텀 쿼리 지원

------------------------------------------------------------------------

## 4. 구조

Controller → Service → Repository → JPA → DB

------------------------------------------------------------------------

## 5. 장점

-   생산성 향상
-   유지보수 용이
-   테스트 친화적
-   확장성

------------------------------------------------------------------------

## 6. 활용

기업 시스템, 그룹웨어, ERP 등에 적합

------------------------------------------------------------------------

## 7. 정리

Spring Data는 백엔드 필수 기술이다.
