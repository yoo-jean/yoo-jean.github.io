# Spring 면접 질문 & 모범 답변 정리

본 문서는 Spring 백엔드 개발자 면접에서 자주 출제되는 질문과 모범 답변을
정리한 자료이다.

실무 경험과 함께 활용하면 면접 대비에 큰 도움이 된다.

------------------------------------------------------------------------

## 1. Spring 기본 개념

### Q1. Spring Framework의 가장 큰 장점은?

**A.** DI와 AOP를 기반으로 객체 간 결합도를 낮추고, 유지보수성과
확장성을 높인 구조이다.

------------------------------------------------------------------------

### Q2. DI란 무엇인가요?

**A.** 객체가 직접 의존 객체를 생성하지 않고, 스프링 컨테이너가 대신
주입하는 방식이다.

------------------------------------------------------------------------

### Q3. 생성자 주입을 권장하는 이유는?

**A.** 불변성 보장, 필수 의존성 강제, 테스트 용이성 때문이다.

------------------------------------------------------------------------

## 2. Bean / Scope

### Q4. 스프링 빈이란?

**A.** 스프링 컨테이너가 생성·관리하는 객체이다.

------------------------------------------------------------------------

### Q5. 싱글톤 스코프 주의점은?

**A.** 필드에 상태를 저장하면 동시성 문제가 발생한다.

------------------------------------------------------------------------

### Q6. 프로토타입은 언제 사용하는가?

**A.** 요청마다 새로운 객체가 필요할 때 사용한다.

------------------------------------------------------------------------

## 3. AOP / 트랜잭션

### Q7. @Transactional 동작 원리는?

**A.** AOP 기반 프록시가 트랜잭션을 제어한다.

------------------------------------------------------------------------

### Q8. 트랜잭션이 적용되지 않는 경우는?

**A.** 같은 클래스 내부 호출(Self Invocation) 시 적용되지 않는다.

------------------------------------------------------------------------

### Q9. AOP 사용 사례는?

**A.** 로그, 보안, 트랜잭션, 성능 측정 등이다.

------------------------------------------------------------------------

## 4. JPA / Spring Data

### Q10. Spring Data JPA 장점은?

**A.** Repository 자동 구현과 CRUD 제공으로 생산성을 높인다.

------------------------------------------------------------------------

### Q11. N+1 문제란?

**A.** 연관 조회 시 쿼리가 반복 실행되는 문제이다.

------------------------------------------------------------------------

### Q12. N+1 해결 방법은?

**A.** fetch join, EntityGraph, DTO 조회 방식이다.

------------------------------------------------------------------------

## 5. Controller / 요청 처리

### Q13. @RequestBody vs @ModelAttribute 차이?

**A.** Body는 JSON, ModelAttribute는 파라미터 바인딩이다.

------------------------------------------------------------------------

### Q14. REST URL 설계 기준은?

**A.** 리소스 중심 설계, Path는 식별자, Param은 조건이다.

------------------------------------------------------------------------

## 6. 성능 / 운영

### Q15. 성능 개선 경험은?

**A.** 쿼리 튜닝과 인덱스 최적화로 응답 속도 개선 경험이 있다.

------------------------------------------------------------------------

### Q16. 장애 대응 방법은?

**A.** 로그 분석 → 원인 파악 → 개선 → 재발 방지 순이다.

------------------------------------------------------------------------

## 7. 구조 설계

### Q17. Controller / Service / Repository 역할은?

**A.** 요청, 비즈니스 로직, 데이터 접근 역할이다.

------------------------------------------------------------------------

### Q18. Service에 트랜잭션을 거는 이유는?

**A.** 비즈니스 단위 관리 목적이다.

------------------------------------------------------------------------

## 8. 자주 나오는 함정 질문

### Q19. 필드 주입을 지양하는 이유는?

**A.** 테스트 어려움과 순환 참조 위험 때문이다.

------------------------------------------------------------------------

### Q20. Bean 생성 시점은?

**A.** 기본적으로 컨테이너 시작 시 생성된다.

------------------------------------------------------------------------
